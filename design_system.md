# 퓨처나우 플랫폼 — design_system.md

> 지휘부(설계자+AI)가 시안까지 확정한 디자인 시스템이다. 클로드코드는 이 문서를 따라
> 공용 UI·응답 위젯(B①)·리포트 시각화(B③)·참여 진입 흐름·코치 콘솔을 구현한다.
>
> 범위: **색 토큰 · 타이포 · 응답 위젯 5종 · 리포트 시각화 5종 · 종합 배치 · 참여 진입 흐름 · 코치 콘솔.**
> 참조: architecture.md §10·§8·§5.2. ADR-11·12·20.
> 버전: **v3** (응답·리포트 + 진입 흐름 + 콘솔 확정)
>
> **갱신 규약**: 이 파일을 받으면 기존 design_system.md를 **덮어쓴다**(새 파일 생성 금지 — 동명 충돌 방지).

## 0. 핵심 원칙

1. **색은 3단으로만 흐른다**(ADR-11). 원천 hex → 역할(semantic) → 컴포넌트. 컴포넌트는 hex·원천 토큰 직접 참조 금지. 색 교체 시 §1.1 한 곳만.
2. **브랜드 토큰은 진단이 주입, 중립·의미 토큰은 코어 소유**(ADR-12). 퓨처나우 = 네이비+골드.
3. **색값은 잠정**이다. 첫 화면 확정 후 재평가(§1.1 hex 몇 줄).
4. **참여자 화면엔 경고색을 쓰지 않는다.** danger/warning/care 의미색은 **인도자 리포트·콘솔에서만**.
5. **명명은 리포트·콘솔에서만.** 측정값('활력 8')이 강의 어휘('시들음')로 명명되는 것은 인도자 화면뿐. 문항·채점 출력엔 구인 식별자만.
6. **돌봄 우선.** 인도자 화면(리포트·콘솔)은 "먼저 챙길 분"을 늘 위에 둔다. 관리 도구가 아니라 돌봄 도우미.

## 1. 색 토큰 — 3단 구조

### 1.1 원천 팔레트 (1차 — 색 교체 시 여기만)

```css
:root {
  /* 브랜드 — 네이비 (잠정) */
  --navy-900:#0F1B2D; --navy-700:#1B2A41; --navy-500:#34466A; --navy-300:#5B6B8C;
  /* 브랜드 — 골드 (잠정) */
  --gold-700:#B8923D; --gold-500:#C9A24B; --gold-300:#D9B96E; --gold-100:#EFE2C2;
  /* 중립 (warm gray) */
  --gray-0:#FFFFFF; --gray-50:#F7F6F4; --gray-100:#EEECE8; --gray-200:#DEDAD3;
  --gray-400:#A8A29A; --gray-600:#6B655C; --gray-800:#3A352F; --gray-900:#211E1A;
  /* 의미색 — 인도자 화면 전용 (참여자 응답 화면 사용 금지) */
  --green-600:#2E7D6B; --amber-600:#C0852E; --red-600:#B23B3B;
  /* 의미색 — 리포트·콘솔용 저채도 톤 (절제 표시) */
  --care-fill:#FAECE7;  --care-line:#D85A30;  --care-text:#993C1D;
  --thrive-soft:#9FE1CB; --mid-soft:#E9D8A6;  --languish-soft:#F0997B;
}
```

### 1.2 역할 토큰 (2차 — 라이트). 컴포넌트는 이것만 참조

```css
:root {
  --color-primary:var(--navy-700);      --color-primary-strong:var(--navy-900);
  --color-accent:var(--gold-500);       --color-accent-soft:var(--gold-100);
  --color-bg:var(--gray-0);             --color-surface-1:var(--gray-50);
  --color-surface-2:var(--gray-0);      --color-surface-sunken:var(--gray-100);
  --color-border:var(--gray-200);       --color-border-strong:var(--gray-400);
  --color-text:var(--gray-900);         --color-text-secondary:var(--gray-600);
  --color-text-muted:var(--gray-400);   --color-text-on-accent:var(--gray-0);
  --color-success:var(--green-600); --color-care:var(--amber-600); --color-danger:var(--red-600);
}
```

### 1.3 다크 토큰 (2차 — 다크). 역할만 재지정, 컴포넌트 무변경

```css
@media (prefers-color-scheme:dark){:root{
  --color-primary:var(--gold-500);      --color-primary-strong:var(--gold-300);
  --color-accent:var(--gold-500);       --color-accent-soft:var(--navy-700);
  --color-bg:var(--gray-900);           --color-surface-1:var(--gray-800);
  --color-surface-2:#2A2722;            --color-surface-sunken:var(--gray-900);
  --color-border:#3D3833;               --color-border-strong:var(--gray-600);
  --color-text:var(--gray-50);          --color-text-secondary:var(--gray-400);
  --color-text-muted:var(--gray-600);   --color-text-on-accent:var(--navy-900);
}}
```

### 1.4 규약
- 컴포넌트 CSS·className은 **2차 역할 토큰만**. `#`hex·`--navy-*`·`--gold-*` 직접 참조 금지.
- 선택(selected) = `--color-accent`(골드). 색 교체는 §1.1 원천만 수정.

### 1.5 색 역할 모델
> **네이비 = 앱의 틀 / 골드 = 참여자의 흔적.**
- **네이비**(`--color-primary`): 헤더·화면 제목·블록 헤딩·주요/보조 버튼.
- **골드**(`--color-accent`): 선택(도트·세그먼트·체크)·진행바·슬라이더 값·OTP 입력 중 칸·차수명 부제.
- 원천 hex 무변경, 역할 배정만. 다크에선 primary가 골드로 올라옴.

## 4. 응답 위젯 5종 (B① 렌더 사양) · **참여자 화면**

공통: 노출 문자열은 prompt·title·intro·label·양극 레이블뿐. 선택색 골드. 위젯 모양=문항 성격(세그먼트=방향/도트=강도/슬라이더=거리). **경고색 배제.**

### 4.1 나침반 — 양극 세그먼트 바 (bipolar 5점)
- 5칸 연결 세그먼트, 중앙 유지. 선택 칸 골드. 칸 높이 ≥46px.
- 양극 레이블: 컨트롤 위, 좌/우 각 ~44% 폭 블록 분리(좌·우 정렬, 2줄 허용, 중앙 여백). 한 줄 배치 금지.

### 4.2 지금의 나 — 리커트 (likert 5점) · 2단 구조
- 행 = 질문 한 줄(위) → 5원 전체 폭(아래). 좌우 분할 금지.
- 척도 헤더(min·center·max) 블록 상단 sticky 1회. 헤더·도트 동일 전체 폭 5등분 → min↔1·center('보통')↔3·max↔5 정렬.
- 도트 22px·탭타깃 44px·선택 골드. centerLabel 없으면 중앙 생략(ADR-20).

### 4.3 간격 — 슬라이더 + 숫자 (numeric 0~10)
- 0~10 슬라이더 + 옆 큰 숫자(accent, tabular-nums). 5영역 한 화면 세로 스택.

### 4.4 주관식 — 텍스트 영역 (text)
- 한 문장 2줄·~200자 / 여러 줄 4줄·~500자. 입력 배경 surface-sunken. placeholder 존대체. 글자수 우하단 micro.

### 4.5 체크 — 행 토글 (check)
- 텍스트(좌) + 체크박스(우측, 세로 중앙). 카드 전체 탭 타깃. 경고색·느낌표 금지. 선택 시 행 골드 소프트·박스 골드. 돌봄 체크도 동일한 담담한 외양.

## 5. 리포트 시각화 5종 (B③ 렌더 사양) · **인도자 화면**

여기서 의미색이 켜지고 측정값이 강의 어휘로 명명된다. 단 절제 — 낮은 값은 낙인이 아니라 돌봄 신호. 본문은 네이비·회색, 의미색은 돌봄 신호 한 곳에만.

### 5.1 나침반 이동 — 덤벨 (4축)
- NAV1~4 가로 트랙(좌극↔우극). 사전=빈 점(회색), 사후=찬 점(네이비), 연결선. 오른쪽 이동=강화. 다이얼은 장식용만.

### 5.2 간격 — 레이더 (5영역)
- 오각. 사후=면 채움(네이비 13%+실선), 사전=점선 윤곽(회색). 정밀 수치는 영역별 덤벨/표 보조.

### 5.3 GROW+F — 충전 막대 (5축)
- G·R·O·W·F 가로 막대, 사전(회색)·사후(네이비) 나란히. 가장 짧은 사후=보강 포인트. 그룹 한 시점은 단계 사슬(원 채도=충전도).

### 5.4 활력 지수 — 띠 이동 (5~25) · **구간 확정**
- 구간: **시들음 ≤10**(`--languish-soft`) / **중간 11~17**(`--mid-soft`) / **번성 18~25**(`--thrive-soft`).
- 사전→사후 화살표 이동. 옆/아래 상태 배지 + 돌봄 안내. 명명은 여기서. 사후 없으면 단일 구간 게이지 폴백.

### 5.5 돌봄 신호 — 조건부 배너
- 활력 시들음 OR Red Flag OR 돌봄 체크 시에만 최상단 저채도 띠(`--care-*`). 우선순위 안내("돌봄 권장 · 가벼운 안부를 권합니다"). 경보·낙인 아님. 신호 없으면 배너 미렌더.

## 6. 종합 리포트 배치 (B③ 개인 리포트)

```
[네이비 헤더] 이름 · 사전·사후 비교 · 인도자 전용
[돌봄 신호]   조건부. 시들음·Red Flag·돌봄 체크 시에만
[헤드라인]    활력 띠 이동  |  나침반 덤벨        ← "달라졌는가"
[깊이]        간격 레이더   |  GROW+F 충전막대    ← "어디가 자랐고 약한가"
[그 사람의 말] E1~E3 주관식 · 돌봄 메모
```
- 데스크톱 2×2, 모바일 1열. 위계 동일. 의미색은 돌봄 배너에만. react-pdf도 동일 위계.

## 7. 참여 진입 흐름 (참여자 화면)

```
① 코드 입력 → ② 차수 미리보기 → [로그인/가입] → ③ 시작 안내 → 응답
```

### 7.1 코드 입력
- **5칸 분할(OTP 스타일)**, 한 칸당 1글자. 입력 중 칸 골드 테두리, 빈 칸 회색. 5칸 미완성 시 확인 버튼 비활성. 헤더 네이비.

### 7.2 차수 미리보기 (CohortPreview)
- `resolve_cohort_by_code` 메타 표시: 차수명(네이비 크게)·인도자명·현재 인원·진단 종류·예상 시간. 민감정보(응답·명단·전화) 미노출.
- "이 모임에 들어갑니다" + [들어가기](네이비)·[아니에요](테두리). 비로그인으로 표시 가능(SECURITY DEFINER 메타).
- 이 메타용 별도 타입 **CohortPreview**(coach_name·member_count 포함, Cohort 도메인 밖) — 계약 추가 시 협의.

### 7.3 로그인/가입 (② 다음)
- '들어가기' 시점에 인증 필요(enrollByCode가 auth.uid() 요구). 신규=가입·기존=로그인 한 화면 분기.
- 가입 최소화: 이메일+비밀번호 또는 소셜. **이름·전화 강요 금지**(ADR-03 참여자 선택). 이후 enrollByCode.

### 7.4 시작 안내
- 호흡 고르기: "정답 없음·떠오르는 대로·중간에 멈춰도 됨"(존대체). + **보안 고지**("여기 적는 모든 것은 인도자 한 사람만 봅니다") 회색 박스. 헤더 부제에 차수명 골드.
- **동의는 버튼=동의** + 보안 고지로 갈음. 명시적 동의 체크박스 없음(신뢰 기반 진입). 개인정보 수집 확대 시 명시 동의 추가(plan.md).

## 8. 코치·운영자 콘솔 (인도자 화면)

원칙: 코치는 코칭 전문가이나 시스템 초보자(architecture §1). **헤매지 않게 — 기능 밀도가 아니라 "지금 할 일"이 먼저.** 돌봄 우선(§0.6).

### 8.1 콘솔 홈 — 돌봄·할 일 중심
- 진입 기본 화면. 위계: [네이비 헤더] → **[먼저 챙길 분]**(저채도 코랄, 이름+사유, 탭→개인 리포트) → [진행 중 차수 요약](진행바·응답수·리포트 보기) → [모든 차수]·[+ 새 차수] 버튼.
- 차수 목록 전체는 '모든 차수' 뒤(§8.4). 자주 쓰는 건 홈, 가끔 쓰는 건 한 단계 안.

### 8.2 차수 개설 — 3스텝 + 완료
- ① 차수 정보(이름, "참여자에게 보이는 이름" 안내) → ② 진단 종류(사전/종료 카드 선택)·정원(−/+ 스테퍼) → ③ 완료·코드 공유.
- 진행바·"n/3 단계"로 부담 완화. 한 화면 한 질문.
- **③ 코드 공유가 핵심**: 코드 네이비 박스 크게 + [코드 복사]·[초대 링크 공유] + **그대로 전달할 안내 문구 예시**(회색 박스). 코드는 자동 생성(cohorts.code, 5자리 혼동글자 제외).

### 8.3 차수 상세 — 돌봄 우선 명단
- [네이비 헤더(←·차수명·상태)] → [3숫자 요약: 응답 완료(네이비)·대기(회색)·돌봄(코랄)] → 명단 3묶음:
  - **먼저 챙길 분**(저채도 코랄 카드, 이름+사유) — 맨 위.
  - **응답 완료**(회색) — 각자 우측 `›`로 개인 리포트 진입.
  - **아직 안 함**(흐림, 미응답) — 리포트 없음(화살표 없음).
- 하단 [참여 코드 + 다시 공유]. 마감·정원 조정 등 덜 쓰는 관리는 헤더 메뉴 안.
- 의미색은 돌봄(상단 숫자·먼저 챙길 분)에만.

### 8.4 모든 차수 — 차수 목록
- '모든 차수'에서 진입. 헤더에 ←(홈 복귀). 차수 카드 = §8.1 홈 요약 카드와 동일 양식(진행 상태·응답률·돌봄 수). 카드 탭 → 차수 상세(§8.3).

### 8.5 그룹 리포트 (선택)
- 차수 상세에서 진입. 그룹 평균: GROW+F 단계 사슬(§5.3) + 간격 그룹 평균 레이더. 1주차 오프닝용.

### 9.1 리포트 차트군 — **인스트루먼트 소유** (정정)
- Dumbbell·Radar·ChargeBars·VitalityBand·CareBanner는 **퓨처나우 전용**이므로 `/src/instruments/futurenow/report/`에 둔다. **코어에 두지 않는다.**
- 근거: 이 차트들은 퓨처나우 측정 구조(4축·5영역·GROW+F·활력)에 묶임. SAIL은 2축이라 무의미. ADR-01(리포트 엔진 비공유)·코어 경계(진단은 코어 미참조) 준수.
- 단 저채도 의미색 토큰(`--care-*`·`--*-soft`)은 globals.css 공용 유지(토큰은 코어, 차트는 인스트루먼트).
- (v2 §7의 "리포트 차트군 → 코어 /src/core/ui" 기술은 오기였음. 본 절로 정정.)
