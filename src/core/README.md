# /src/core — 1층 공유 런타임 (C-ready)

코어는 **채점하지 않는다.** 신원·계정 · 차수·코치·가입 · 응답봉투 · 알림·돌봄 ·
공용 UI를 *운영*하는 런타임(OS)이다. 채점·리포트·진단 고유 로직은 한 줄도 두지 않는다.
(architecture §2 핵심원리 1 / CLAUDE §2)

## 경계 규율 (CLAUDE §1 · architecture §4)

- `/core` 는 `/instruments` 를 **참조하지 않는다.**
- 코어가 바라보는 것은 오직 `/contracts` 뿐이다.
- 이 한 겹이 나중에 레포를 갈라도(C안 패키지 추출) 계약이 깨지지 않게 막는다.

## 하위 경계

| 폴더 | 역할 |
|---|---|
| `auth` | 인증·세션·현재 사용자 |
| `identity` | users·role·신원 필수성 정책 강제 |
| `cohort` | 차수·가입코드·참여(enrollment)·코치 승인 |
| `response` | 응답 봉투 저장/조회 · 응답 러너(ResponseRunner) |
| `alert` | 알림 전달·돌봄 명단 |
| `ui` | 공용 컴포넌트 (디자인 시스템 확정 후 — CLAUDE §8) |

> 향후 C안 추출 시 이 디렉터리가 독립 패키지(또는 워크스페이스)가 된다(plan.md §1).
