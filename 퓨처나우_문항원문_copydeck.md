# 퓨처나우 문항 원문 (Copy Deck) — flow.ts / copy.ts 반영용

> 출처: 확정된 사전·종료 자기진단지. **검증된 콘텐츠이므로 임의 수정·윤문 금지.**
> 1~28번 문항(prompt·양극 레이블)은 사전·종료 공용. 블록 서사(intro)와 E1~E3·들어가며는 wave별로 다르다.
> 코드는 architecture §9.1 매핑 그대로. 화면 노출 문자열은 prompt·intro·레이블뿐(코드·구인·STEP 비노출, §9.4).

---

## 블록 0 — 들어가며 (INTRO, text·multiline)

- **사전(pre) intro**: "진단을 시작하기 전에, 잠시 미래의 한 장면을 떠올려 봅니다.\n5년 뒤, 당신은 어떤 사람이 되어 있고 싶습니까?\n정답을 찾지 마세요. 떠오르는 장면, 표정, 그가 건네는 한마디 — 무엇이든 좋습니다. 다듬지 않아도 됩니다."
  - placeholder: "[ 나의 인생 조감도 — 한 문장 스케치 ]"
- **종료(post) intro**: "진단을 시작하기 전에, 이 여정을 막 시작하던 5주 전의 당신을 떠올려 봅니다.\n5주 전, 막막함 속에 첫 페이지를 펼쳤던 그 사람에게 지금의 당신은 무슨 말을 건네고 싶습니까?\n정답을 찾지 마세요. 그가 가장 듣고 싶어 했을 한마디 — 무엇이든 좋습니다. 다듬지 않아도 됩니다."
  - placeholder: "[ 5주 전의 나에게 — 한마디 ]"

## 블록 1 — 나의 나침반 (NAV1~4, bipolar 5점)

- **사전 intro**: "양 끝 중 어느 쪽이 옳은 것은 아닙니다. 요즘의 나에게 더 가까운 쪽으로 점을 표시해 주세요."
- **종료 intro**: "5주 전과 같은 질문입니다. 그때를 기억하려 말고, 지금의 나에게 더 가까운 쪽으로 점을 표시해 주세요."

| 코드 | prompt | 좌(1) leftLabel | 우(5) rightLabel |
|---|---|---|---|
| NAV1 | 새로운 일을 앞두고 내가 먼저 떠올리는 것은 | 잘못될 경우와 그에 대한 대비책 | 잘되었을 때의 장면 |
| NAV2 | 잘 풀리는 또래의 소식을 들으면 나는 | 나도 모르게 내 처지와 견주어 보게 된다 | 부럽긴 해도 금세 내 일로 돌아온다 |
| NAV3 | '나'라는 사람을 가장 잘 설명해 주는 것은 | 지금까지 걸어온 길 | 앞으로 걷고 싶은 길 |
| NAV4 | 큰 방향을 바꿔야 할 순간이 오면, 나는 | 그동안 쌓아 온 것이 소중하여 쉽게 바꾸기 어렵다 | 오늘 새로 시작하듯 백지에서 다시 고를 수 있다 |

## 블록 2 — 지금의 나 (A·C·D 코드, likert 5점)

- **사전 intro**: "각 문장이 요즘의 나에게 얼마나 맞습니까? 깊이 생각하지 말고, 떠오르는 대로 표시해 주세요."
- **종료 intro**: "5주 전과 같은 문장들입니다. 그때의 답은 잊고, 지금 이 순간의 나를 그대로 표시해 주세요."
- **척도 레이블**: minLabel "전혀 아니다" · (중앙) "보통" · maxLabel "매우 그렇다"

| 코드 | prompt |
|---|---|
| A1 | 아침에 눈을 뜰 때, 오늘 하루가 기대된다. |
| C3 | 나는 내 시간과 에너지가 실제로 어디에 쓰이고 있는지 정확히 알고 있다. |
| A2 | 열심히 살고는 있지만, 제자리걸음이라는 느낌이 든다. |
| C6 | 누가 묻는다면 '지금 내 인생에서 가장 중요한 단 하나'를 바로 답할 수 있다. |
| D1 | 마음속에만 몇 년째 품고 있는, 아직 시작하지 못한 일이 있다. |
| C2 | 나는 5년 뒤 내가 어떤 사람이 되어 있을지 구체적인 장면으로 그릴 수 있다. |
| A5 | 잠들기 전, 생각보다 오래 휴대폰을 붙잡고 있다가 자는 날이 많다. |
| C8 | 내 주변에는 내 목표를 알고 정기적으로 점검해 주는 사람이 있다. |
| C5 | 해야 할 일이 많아, 정작 중요한 일이 자주 뒤로 밀린다. |
| A3 | 요즘 시간 가는 줄 모르고 빠져드는 일이 있다. |
| D2 | "조금 더 준비되면 시작해야지"라는 말을, 작년에도 했고 올해도 하고 있다. |
| C1 | 지난 실패담을 꺼낼 때, 그 일이 내게 남긴 것까지 함께 말하게 된다. |
| A4 | 특별히 불행하지는 않지만, 살아 있다는 생생함도 느껴지지 않는다. |
| C7 | 최근 1년 사이, 한 달 넘게 이어 간 새로운 다짐이 있다. |
| D3 | 지금 생활에 큰 불만은 없다. 다만 '5년 뒤에도 이대로 좋은가'라는 질문은 좀 피하고 싶다. |
| C4 | 나는 어떤 습관을 버리고 어떤 습관을 새로 만들어야 할지 분명히 알고 있다. |
| C9 | 일이 잘 풀리지 않는 시기에도, 나 자신을 보는 눈은 크게 흔들리지 않는다. |

> 위 표는 **저장 코드 기준 논리 순서**다(A1=문항5 … C9=문항21). 화면 표시 순서는 코어 러너가 제약 무작위 배열(firstPolarity 'positive', maxConsecutiveSameNegative 1)로 매 응답 새로 매긴다. polarity·역채점은 §9.1·§9.3 그대로.

## 블록 3 — 믿음의 자리 (F1·F2, likert 5점, optional)

- **intro(사전·종료 공용)**: "신앙은 사람마다 자리가 다릅니다. 지금의 솔직한 상태를 표시해 주세요. 답하기 어려운 문항은 비워 두어도 괜찮습니다."
- **척도**: minLabel "전혀 아니다" · maxLabel "매우 그렇다"

| 코드 | prompt |
|---|---|
| F1 | 나는 내 삶에 성취 너머의 더 큰 의미가 있다고 믿는다. |
| F2 | 최근 한 달 사이, 기도나 말씀 때문에 결정이나 마음을 바꾼 적이 있다. |

## 블록 4 — 다섯 영역의 간격 (B1~B5, numeric 0~10)

- **사전 intro**: "내가 원하는 모든 것을 이룬 5년 후의 내 모습을 10점이라 할 때, 지금의 나는 각 영역에서 몇 점입니까?\n점수가 낮다고 실망하지 마세요. 그 간격의 크기가 바로, 당신이 지금 그것을 간절히 바라고 있다는 증거입니다."
- **종료 intro**: "5년 후의 완성된 내 모습을 10점이라 할 때, 각 영역에서 지금의 나는 몇 점입니까?\n두 숫자 사이의 거리가, 5주 동안 당신이 실제로 움직인 만큼입니다. 작은 변화라도 그 방향이 중요합니다."
  - 종료 화면은 '오늘' 점수만 수집한다. '5주 전' 값은 페어링된 사전 응답에서 가져와 나란히 보여준다(별도 입력 아님).

| 코드 | label(prompt) |
|---|---|
| B1 | 일 (Work) — 성취와 보람 |
| B2 | 재정 (Wealth) — 경제적 안정 |
| B3 | 관계 (Relationships) — 사람들과의 친밀함 |
| B4 | 건강 (Health) — 신체적·정신적 활력 |
| B5 | 기여 (Contribution) — 타인을 향한 헌신 (이웃·공동체를 향한 섬김과 나눔) |

## 블록 5 — 나에게 묻는 시간 (E1~E3 text + 돌봄 체크)

- **사전 intro**: "정답이 없는 질문입니다. 떠오르는 대로, 솔직하게 적어 주세요."
- **종료 intro**: "정답이 없는 질문입니다. 5주를 천천히 되감아 보며 적어 주세요."

| 코드 | 사전 prompt | 종료 prompt |
|---|---|---|
| E1 | 이 세미나가 끝났을 때, 당신의 무엇이 달라져 있기를 바랍니까? | 5주 전, 당신은 '이 세미나가 끝나면 무엇이 달라지길' 바랐습니다. 그 바람은 지금 어떻게 되었습니까? |
| E2 | '들어가며'에서 5년 뒤의 나를 떠올렸을 때, 어떤 느낌이 올라왔습니까?\n설렘, 막막함, 두려움, 그리움 … 무엇이든 좋습니다. | 5주 동안 당신 안에서 가장 크게 달라진 것 하나를 꼽는다면 무엇입니까?\n아주 작은 변화여도 좋습니다. 그것이 가장 진짜일 때가 많습니다. |
| E3 | 이번 세미나에서 인도자에게 바라는 점이 있다면 자유롭게 적어 주세요. (선택) | 이 여정은 끝이 아니라 시작입니다. 앞으로 계속 붙잡고 싶은 단 하나는 무엇입니까? |

- **돌봄 체크 (CARE, check)**
  - 사전 label: "세미나 참석과는 별도로, 인도자에게 1:1 코칭/상담을 받고 싶습니다."
  - 종료 label: "세미나 종료 후에도, 인도자에게 1:1 코칭/상담을 이어 가고 싶습니다."

## subjectProfile 필드 (응답 시점 스냅샷)

진단지 하단 수집 항목. 실명은 코어 `users.name` 재사용, 전화는 코어 `user_contacts`. 아래만 subjectProfile에 둔다.

| 필드(클로드1 명명) | 화면 label | 사전 | 종료 |
|---|---|---|---|
| ageBand | 연령대 | ○ | ○ |
| faithYears | 신앙 연수 | ○ | ○ |
| writtenAt | 참여 계기(사전) / 작성일(종료) | "참여 계기" | "작성일" |
| motivation | (사전의 '참여 계기'를 motivation으로 둘지 writtenAt과 분리할지 — 아래 지시 참조) | | |

> 주의: 사전엔 "참여 계기", 종료엔 "작성일"이 하단에 있다. 두 wave가 필드가 다르므로, `writtenAt`(작성일)과 `motivation`(참여 계기)을 **분리**하고 wave별로 노출 필드를 다르게 두는 것을 권한다.

## 안내 문구 (화면 상단·하단 공용 카피)

- 보안 고지(전 화면 공통): "여기 적는 모든 것은 세미나 인도자 한 사람만 봅니다."
- 사전 도입 안내: "정답은 없습니다. 그냥 '지금의 나'를 만나는 시간입니다."
- 종료 도입 안내: "5주 전과 똑같은 질문입니다. 그때의 나와 만나는 시간입니다."
